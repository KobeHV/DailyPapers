## Memory & Long Context

### 1. EverMemOS: A Self-Organizing Memory Operating System for Structured Long-Horizon Reasoning
- **作者**: Chuanrui Hu, Xingze Gao, Zuyi Zhou, Dannong Xu, Yi Bai, Xintong Li, Hui Zhang, Tong Li, Chong Zhang, Lidong Bing, Yafeng Deng
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2601.02163) | [PDF](https://arxiv.org/pdf/2601.02163)
- **来源**: 🔥 HF Trending，⬆️ 13

- **核心方法**: 为 LLM 设计了一套仿生记忆操作系统，分三步走：(1) Episodic Trace Formation——把对话流转化为 [[MemCell]]（包含情景痕迹、原子事实、有时限的 Foresight 信号）；(2) Semantic Consolidation——把 MemCell 组织成 [[MemScene]] 主题簇，蒸馏出稳定语义结构并更新用户画像；(3) Reconstructive Recollection——用 MemScene 引导 Agent 检索，拼出推理所需的最小充分上下文。
- **对比方法/Baselines**: 在 [[LoCoMo]] 和 [[LongMemEval]] 上对标，报告 SOTA。但对比的 baselines 都是些什么水平没细说。
- **借鉴意义**: Memory 系统设计思路可参考 MemCell→MemScene 的分层抽象，但整篇论文偏系统设计，对做 LLM 训练的人直接帮助有限。
- **锐评**: 名字起得很大——"Memory Operating System"，实际就是把记忆分了两个粒度（cell + scene）做检索。Engram 这个概念从神经科学借来唬人，但具体实现没看出和现有 hierarchical memory 有什么本质区别。LoCoMo 上号称 SOTA 但只拿了 13 个 upvote，社区用脚投票了。⚠️

### 2. TF-Engram: A Train-Free Engram with SSD-Backed Memory for Large Language Models
- **作者**: Yutang Ma, Kecheng Huang, Xikun Jiang, Zili Shao
- **机构**: 北京理工大学珠海, 香港中文大学
- **链接**: [arXiv](http://arxiv.org/abs/2607.07388v1) | [PDF](https://arxiv.org/pdf/2607.07388v1)
- **来源**: 📄 arXiv 关键词检索

![](https://arxiv.org/html/2607.07388v1/figures/tech_overview2.png)

- **核心方法**: 提出免训练的 [[Engram]] 记忆系统。离线阶段从外部语料构建 phrase-specific 语义记忆表，存储在 GPU→DRAM→SSD 三级层次中；推理时用 Early-Exit Guided Predictive Prefetching 隐藏外部存储延迟。关键卖点：不做 hash 压缩（避免了现有 engram 方法的 phrase 碰撞问题），完全免训练。
- **对比方法/Baselines**: Qwen3-0.6B 上平均下游分数从 57.6 提升到 59.4，超过同等参数的 [[LoRA]] 微调基线。
- **借鉴意义**: 对于想在推理阶段外挂知识但不想重新训练的场景有参考价值，SSD 层级的设计对资源受限部署也有启发。
- **锐评**: 思路是对的——hash 压缩确实会丢语义，换成精确匹配的 phrase table 是合理的改进。但提升幅度太寒碜了：57.6→59.4，在 Qwen3-0.6B 这种小模型上才涨了不到 2 个点。SSD 延迟怎么掩盖的也没给出足够说服力的 ablation。想法好，执行不够狠。👀

### 3. MemDefrag: Latent Memory Defragmentation for Large Language Models
- **作者**: Ruiyi Yan, Zhuoyuan Mao, Yiwen Guo
- **机构**: 未知
- **链接**: [arXiv](http://arxiv.org/abs/2607.05969v1) | [PDF](https://arxiv.org/pdf/2607.05969v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 针对 [[MemoryLLM]] 和 [[M+]] 这类 latent memory 方案在更新记忆时性能退化的问题，发现中间 transformer 层的注意力密度始终集中在目标记忆片段上——利用这个天然存在的"追踪信号"做记忆碎片整理（排序、重排、过滤），超出容量时用信息量引导的比例遗忘机制。
- **对比方法/Baselines**: 50 次记忆更新后知识保持 43.0%，vs MemoryLLM 的 17.4% 和 M+ 的 17.6%。
- **借鉴意义**: "中间层注意力密度最高"这个发现本身就有价值——给 latent memory 系统的设计提供了一个无需训练的定位信号。
- **锐评**: 今天四篇 memory 论文里最有干货的一篇。不是又造一个轮子，而是在现有 latent memory 方案上发现了一个具体的失败模式（位置编码错位 + 无法定位目标片段），然后针对性地解决了。43% vs 17% 的提升是实打实的。可惜没开源代码，实验也只在 MemoryLLM 和 M+ 两个基准上跑。需要看全文确认方法泛化性。👀

### 4. Mem0: Building Production-Ready AI Agents with Scalable Long-Term Memory
- **作者**: Prateek Chhikara, Dev Khant, Saket Aryan, Taranjeet Singh
- **机构**: 未知
- **链接**: [arXiv](https://arxiv.org/abs/2504.19413) | [PDF](https://arxiv.org/pdf/2504.19413)
- **来源**: 🔥 HF Trending，⬆️ 67

- **核心方法**: 一个面向生产的记忆架构，动态提取、整合和检索对话中的关键信息。另有增强版 Mem0^g 用图结构捕获对话元素间的关系。记忆操作包括 ADD、UPDATE、CORRECT、DELETE、NOOP 五类。
- **对比方法/Baselines**: 在 [[LOCOMO]] 基准上对比 6 类 baseline（RAG、全上下文、[[MemGPT]]、[[LangMem]]、[[ReadAgent]] 等），LLM-as-Judge 指标比 OpenAI 方案高 26%，p95 延迟降低 91%。
- **借鉴意义**: 工程实现参考——延迟和 token 节省数据对生产部署有实际参考价值。
- **锐评**: 这是一篇产品说明书，不是研究论文。"比 OpenAI 方案高 26%"这种话术经典——拿自己的定制方案跟别人的通用 API 比，不公平。图记忆增强版只比基础版高 2%，说明图结构在这个场景下作用有限。67 个 upvote 说明市场需求存在，但学术贡献约等于零。💤

### 5. HiLS-Attention: Hierarchical Landmark Sparse Attention — Toward Infinite Context Modeling
- **作者**: Xiang Hu, Xinyu Wei, et al. (Tencent HY Team)
- **机构**: Tencent, ShanghaiTech University, UCSD, HKUST
- **链接**: [arXiv](https://arxiv.org/abs/2607.02980) | [PDF](https://arxiv.org/pdf/2607.02980)
- **来源**: 🔥 HF Trending，⬆️ 52

![](https://arxiv.org/html/2607.02980v1/x1.png)

- **核心方法**: 提出分层稀疏注意力，让 chunk 选择在 LM loss 下端到端学习。每个 query 独立与各检索到的 chunk 做 attention 提取 chunk-specific 信息，输出按 chunk 检索分数融合。关键创新：检索分数直接参与前向计算，使得 LM loss 能优化检索本身。
- **对比方法/Baselines**: 345M 模型继续训练至 256K 上下文，[[RULER]] 基准上显著超过 [[NSA]] 等方法。可外推 64 倍训练长度且保持 90% 检索准确率。现有 full-attention 模型可通过轻量继续训练转换。
- **借鉴意义**: 端到端学习稀疏注意力的思路值得关注——打破了"先设计稀疏模式再验证"的传统范式。转现成 dense 模型的能力也很实用。
- **锐评**: 今天的 memory/long-context 论文里质量最高的一篇。端到端学习 chunk selection 是真正的创新点——之前的工作（NSA 等）都是人工设计稀疏模式，这篇让模型自己学要关注哪些 chunk。64x 外推 + 90% 检索准确率的数据很漂亮。腾讯 HY 团队的工程能力有保障，代码已开源。唯一遗憾是只在 345M 小模型上做了充分实验。👀

### 6. LongCrafter: Towards Diverse Long-Context Understanding via Evidence-Graph-Guided Instruction Synthesis
- **作者**: Chenhao Yuan et al.
- **机构**: 未知
- **链接**: [arXiv](http://arxiv.org/abs/2607.06160v1) | [PDF](https://arxiv.org/pdf/2607.06160v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 设计了一个分层任务分类体系（32 种细粒度长上下文理解任务类型），作为全局生成先验。三阶段合成流程：(1) 构建任务对齐的长上下文；(2) 分解为显式证据图（建模跨段落依赖）；(3) 严格基于证据 span 生成 instruction-response 对。
- **对比方法/Baselines**: 在 [[LongBench]]、LongBench v2、[[LooGLE]] 上超过所有 SFT 基线，甚至超过官方 post-trained 模型。在 Qwen2.5-7B 和 LLaMA-3.1-8B 上均有效。
- **借鉴意义**: 证据图驱动的数据合成方法论可迁移到其他需要可验证推理数据的场景。位置鲁棒的证据定位能力对长上下文训练有帮助。
- **锐评**: 长上下文 SFT 数据合成这个方向确实缺一篇系统性的工作。32 种任务类型的分类体系设计得挺全面，证据图约束保证了 faithfulness——这一点比 [[LongMagpie]]、[[LongReward]] 等只看表面的方法有本质提升。"lost in the middle"问题的缓解效果不是吹的，有 attention mass 可视化作证。但只验证了 7B/8B 规模，大模型上是否有效存疑。👀

### 7. Improving LLM-Generated Process Model Quality Through Reinforcement Learning: The Role of Reward Function Design
- **作者**: Alexander Rombach, Chantale Lauer, Nijat Mehdiyev
- **机构**: German Research Center for Artificial Intelligence (DFKI), Saarland University
- **链接**: [arXiv](http://arxiv.org/abs/2607.06175v1) | [PDF](https://arxiv.org/pdf/2607.06175v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 系统研究 RL reward function 设计对 LLM 生成质量的影响。用 [[GSPO]]（Group Sequence Policy Optimization）在 48 种 reward 配置下训练 Llama 3.1 8B 和 Qwen 2.5 14B。三个发现：(1) RL 显著提升 pragmatic 和 syntactic 质量，输出变异性降低 6 倍以上；(2) 等权 reward 始终优于加权 reward——刻意强调某个维度反而会崩；(3) invalid penalty 对一个模型至关重要，对另一个完全无关。
- **对比方法/Baselines**: 38 个自动评估指标的全面评估框架。
- **借鉴意义**: "等权 reward 优于加权 reward"这个发现对做 RLHF/RLVR 的人是一个实用教训——别自作聪明调 reward weight。invalid penalty 的模型依赖性也值得注意。
- **锐评**: 这篇的贡献不是方法创新，而是提供了一套 reward design 的经验法则。48 种配置的 grid search 虽然暴力但有用。最反直觉的发现——等权优于加权——说明我们目前对 RL reward shaping 的理解还很原始。应用场景窄（BPMN 流程模型生成），但方法论和结论可以泛化。⚠️ 不够 general 但够 solid。

### 8. Single-Rollout Asynchronous Optimization for Agentic Reinforcement Learning
- **作者**: Zhenyu Hou, Yujiang Li et al.
- **机构**: Tsinghua University
- **链接**: [arXiv](https://arxiv.org/abs/2607.07508) | [PDF](https://arxiv.org/pdf/2607.07508)
- **来源**: 📰 HF Daily，⬆️ 1
- 📒 **笔记**: [[SAO]]

![](https://arxiv.org/html/2607.07508v1/x1.png)

- **核心方法**: 提出 [[SAO]]——单 rollout 异步 RL 优化。核心改动：(1) 用 single-rollout 采样替代 [[GRPO]] 的 group-wise 采样，降低 off-policy 效应；(2) 引入 strict double-side token-level clipping（DIS）稳定训练；(3) 可稳定训练 1000 步。已部署到 GLM-5.2（750B-A40B）的训练中。
- **对比方法/Baselines**: [[SWE-Bench]] Verified、[[BeyondAIME]]、[[IMOAnswerBench]] 上一致超越 GRPO 及其变体。在线学习模拟中 single-rollout 策略特别有效。
- **借鉴意义**: 对正在做 LLM RL 的人非常实用——异步 RL 是趋势，但 GRPO 的 group sampling 确实不匹配异步场景。Single rollout + double-side clipping 的配方可以直接试。
- **锐评**: 今天 RL 方向最实用的一篇。GRPO 的 group-wise 采样在异步训练中水土不服是行业共识但没人系统解决，这篇把方案做干净了还部署到了 750B 模型上。DIS clipping 的设计有理论动机（importance sampling 的 variance 控制）不只是调参玄学。唯一的悬念：GLM-5.2 的成功到底有多少归功于 SAO 而不是其他架构改进？毕竟论文作者就是 GLM 团队的人。🔥

### 9. Multi-module GRPO: Composing Policy Gradients and Prompt Optimization for Language Model Programs
- **作者**: Noah Ziems, Dilara Soylu, Lakshya A Agrawal, Isaac Miller, Liheng Lai, Chen Qian, Kaiqiang Song, Meng Jiang, Dan Klein, Matei Zaharia, Karel D'Oosterlinck, Christopher Potts, Omar Khattab
- **机构**: Stanford, UC Berkeley, University of Notre Dame, Zoom, CMU, Contextual AI, Anyscale, MIT
- **链接**: [arXiv](https://arxiv.org/abs/2508.04660) | [PDF](https://arxiv.org/pdf/2508.04660)
- **来源**: 🔥 HF Trending，⬆️ 6

- **核心方法**: 把 [[GRPO]] 扩展到多模块 LM 程序。按模块对 LM 调用分组（跨 rollout），处理变长和中断轨迹。与自动 prompt 优化（[[BetterTogether]]）组合后，在分类、多跳搜索、隐私保护委托任务上平均提升 11%。已开源到 [[DSPy]] 中。
- **对比方法/Baselines**: 对比纯 post-trained LM 和纯 prompt 优化。多跳搜索 [[HoVer]]、隐私保护 [[PAPILLON]] 等任务。
- **借鉴意义**: 如果你在用 DSPy 搭 multi-LM pipeline，这篇直接把 RL 优化集成进去了。模块级别的分组策略解决了"多模块程序怎么联合优化"的实际问题。
- **锐评**: 来自 DSPy 团队，工程味很重但抓住了真问题——现在谁还只用一个 model call？大部分 LLM 应用已经是 multi-call pipeline 了，但 RL 优化还在单模型层面卷。mmGRPO 的模块级分组是自然的扩展思路，11% 的提升在 classification 这种任务上算显著的。但 6 个 upvote 有点惨淡——可能因为太绑定了 DSPy 生态。👀

### 10. Agon: Competitive Cross-Model RL with Implicit Rival Grading of Reasoning
- **作者**: Vladislav Beliaev
- **机构**: 未知
- **链接**: [arXiv](http://arxiv.org/abs/2607.07690v1) | [PDF](https://arxiv.org/pdf/2607.07690v1)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[Agon]]

- **核心方法**: 让两个模型互为评判者。两者同时解同一道题：轮流扮演"起草者"和"挑战者"，起草者写出解，挑战者读完后解题，奖励是"比对方解得更好"。关键：推理质量被隐式评判——不需要过程标签、不需要 reward model。两个模型只需能力相当、行为有差异。推理时也按这个 cascade 部署：一个起草，另一个读了草稿后答题。
- **对比方法/Baselines**: [[DeepMath]] hard split 上用 Qwen3，pass@1 是 [[GRPO]] 的两倍，约 8 倍于未训练的 [[Mixture-of-Agents]]。Qwen3.5 和 [[Gemma]] 4 上也复现了。
- **借鉴意义**: 如果你觉得 GRPO 的"只对最终答案打分"太粗暴但又不想标注过程数据，这篇提供了一个优雅的替代方案——让两个模型互相 judge。比 self-play 更稳定，因为两个模型行为有差异。
- **锐评**: 今天最有创意的 RL 论文。GRPO 的"不判过程只判结果"导致模型学会水字数而不是思考，这是行业公认的痛点。Agon 的解法漂亮：不设计 reward model、不标过程标签，让竞争本身产生隐式质量信号。"draft-and-challenge"的设计让模型必须写出能说服对手的推理——而不是糊弄一个最终答案。DeepMath 上 GRPO pass@1 翻倍的数据很硬。但只做了 1.7B 以下的实验，大模型上的 scalability 是关键。🔥

### 11. RL Post-Training Builds Compositional Reasoning Strategies
- **作者**: Azwar Abdulsalam, Nishil Patel, Andrew Saxe
- **机构**: 未知
- **链接**: [arXiv](http://arxiv.org/abs/2607.07646v1) | [PDF](https://arxiv.org/pdf/2607.07646v1)
- **来源**: 📄 arXiv 关键词检索
- 📒 **笔记**: [[RL Post-Training]]

- **核心方法**: 在完全可控的 rewrite-grammar 环境中研究 RL post-training 的机制。关键发现：RL 不只是放大已有能力，而是通过阶段性组合机制重组 primitive 能力——先强化 primitive reduction，再发现有效的组合过程（sequential composition 压缩有序 primitive 链、parallel composition 在一步内组合独立 primitive）。对比 [[RFT]]（rejection fine-tuning）：RFT 产生大量无效 shortcut，而 RL 把探索集中在有效的可重用结构上。
- **对比方法/Baselines**: RL 解决了 base model 即使加大采样预算也几乎解决不了的 held-out 问题。
- **借鉴意义**: 对理解"RL post-training 到底做了什么"有根本性贡献。不是 magic，是分阶段的组合策略涌现。结论可以指导 RL training 的设计——pretraining 阶段需要把 primitive 组织成 reduction procedure，RL 才能压缩它们。
- **锐评**: 今天的 hidden gem。没有新方法、没有 SOTA，但是回答了一个基础问题：RL post-training 到底是放大已有能力还是创造新能力？答案比你想的复杂——是"重组"。分析方法非常扎实：trace 级别的审计让人能看到组合策略逐步涌现的过程。RFT vs RL 的对比尤其有启发性：区别不在探索量而在选择性。这个结论如果能在大模型上验证，影响力会很大。🔥

### 12. ARIS: Autonomous Research via Adversarial Multi-Agent Collaboration
- **作者**: Ruofeng Yang et al.
- **机构**: Shanghai Jiao Tong University, Shanghai Innovation Institute

### 19. When Does In-Context Search Help? A Sampling-Complexity Theory of Reflection-Driven Reasoning
- **作者**: Yotam Wolf, Noam Wies, Amnon Shashua
- **机构**: The Hebrew University
- **链接**: [arXiv](http://arxiv.org/abs/2607.06720v1) | [PDF](https://arxiv.org/pdf/2607.06720v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 将 in-context search 建模为推理轨迹上的近似推断——base model 定义先验，self-reflection 提供后验更新的反馈。理论分析给出推理时采样复杂度的条件：当反思能可靠地定位早期错误时，in-context search 可以用多项式次尝试解决零样本通过率指数级小的问题；反之，conditioning on past attempts 不比并行采样好。进一步证明这些增益是可学习的：cross-entropy training on search rollouts 能以多项式样本复杂度恢复所需行为。RL with verifiable rewards 的最优策略扩展恰好实现相同的后验重加权规则。
- **对比方法/Baselines**: 在大型 reasoning model 上验证关键定性预测。
- **借鉴意义**: 对理解 deepseek-R1 风格模型"为什么能通过 search 提升推理"提供了第一个扎实的理论框架。结论直接关联 RL post-training 的设计。
- **锐评**: Theory paper，但是那种读了会让你对 empirical 现象有新理解的 theory。把 in-context search 形式化为 posterior inference 是优雅的抽象。"反思必须能定位早期错误"这个条件是 sharp 的——解释了为什么有时候 search 没用（反思质量不够）。和 RL with verifiable rewards 的等价性证明了这是一套统一的框架。Amnon Shashua（Hebrew University + AI21 Labs 创始人）的署名增加了分量。👀

- **作者**: Feng He, Zhenting Wang, Qifan Wang, Qiang Guan, Dongfang Liu, Ruixiang Tang, Qiankun Li
- **机构**: Purdue University, Rutgers University, Imperial College London, Kent State University, Meta AI

## 今日趋势

RL for LLM 正在从"怎么把 RL 用在 LLM 上"进化到"怎么把 RL 用在 LLM 上更好"——SAO 做异步优化、mmGRPO 做多模块扩展、Agon 做跨模型竞争、RL Post-Training 做机制理解。覆盖了工程优化、架构扩展、训练范式和基础理论四个层面，节奏很健康。

Memory 方向四篇扎堆但各说各话——还在"造轮子"阶段，没人给出一个社区公认的 baseline。HiLS-Attention 是今天长上下文方向的亮点。

多 Agent 系统继续卷框架，ARIS 和 AgentScope 各摆各的摊，都在说"我们搭了一个平台"但没人证明"这个平台产出了什么了不起的研究"。

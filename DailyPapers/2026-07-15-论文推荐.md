## 1. GLM-5: from Vibe Coding to Agentic Engineering
- **作者**: GLM-5-Team, Zeng, Aohan, Lv, Xin, Hou, Zhenyu, Du, Zhengxiao 等
- **机构**: Zhipu AI & Tsinghua University
- **链接**: [arXiv](https://arxiv.org/abs/2602.15763) | [PDF](https://arxiv.org/pdf/2602.15763)
- **来源**: 🔥 HF Trending，⬆️ 199
- 📒 **笔记**: [[GLM-5]]

![GLM-5 架构图](https://arxiv.org/html/2602.15763/x1.png)

- **核心方法**: GLM-5 是智谱 GLM 系列的最新模型，预训练 + mid-training 总共使用 28.5T tokens。架构延续 [[MLA]] + [[MoE]] + [[MTP]]（Multi-Token Prediction）组合，延续 GLM-4.5 的技术路线。post-training 分三阶段：SFT → Reasoning RL（类 [[DeepSeek-R1]] 方案）→ Agentic RL（异步 RL 用于 agent 任务）+ General RL。特别引入了 On-Policy Cross-Stage Distillation 来缓解 RL 训练中的 reward hacking。训练框架叫 slime，支持大规模异步 RL。
- **对比方法**: 与 [[GLM-4]]、[[DeepSeek]]、[[GPT-5]] 对标。使用了 [[DSA]]（Dynamic Sparse Attention）和 [[SWA]]（Sliding Window Attention）混合注意力机制。
- **借鉴意义**: 直接对标 DeepSeek-R1 和 GPT-5。其 agentic RL + reasoning RL 的双轨 post-training 策略值得关注，on-policy distillation 的做法对做 RL 训练的人有参考价值。
- **锐评**: 28.5T tokens + MoE + MLA + MTP 这条路线基本是 DeepSeek 趟开的路，GLM-5 做得扎实但没有本质创新。亮点在于 post-training 阶段的 agentic RL——把 agent 能力当做独立训练阶段而不是事后 prompt 工程，这个设计思路比大多数只在 reasoning 上卷的模型聪明。报告里详细讲了异步 RL 的训练框架和 Chinese chip 适配，工程价值大于学术贡献。🔥
- **关联笔记**: [[GLM-5]]（已有笔记）· [[MLA]]（多头潜在注意力）· [[MoE]]（混合专家）· [[DeepSeek LLM]]

## 2. Multi-module GRPO: Composing Policy Gradients and Prompt Optimization for Language Model Programs
- **作者**: Noah Ziems, Dilara Soylu, Lakshya A Agrawal, Isaac Miller, Liheng Lai, Chen Qian, Kaiqiang Song, Meng Jiang, Dan Klein 等
- **机构**: University of Notre Dame, Stanford University
- **链接**: [arXiv](https://arxiv.org/abs/2508.04660) | [PDF](https://arxiv.org/pdf/2508.04660)
- **来源**: 🔥 HF Trending，⬆️ 6
- 📒 **笔记**: [[mmGRPO]]

![Multi-module GRPO](https://arxiv.org/html/2508.04660v2/x1.png)

- **核心方法**: 把 [[GRPO]] 从单个 prompt 扩展到多模块 LM 程序（多个链式/并行的 LM 调用组成的 pipeline）。核心思路是在每个模块级别构建 trajectories 分组，每个模块的 policy 各自优化。同时提出 BetterTogether 策略，把在线 RL 和 prompt 优化组合在一起训练——GRPO 更新模型权重，prompt 优化更新每个模块的指令文本，目标是找到权重和 prompt 的联合最优解。实验在 Banking77、PAPILLON 和 [[HoVer]] 数据集上进行。
- **对比方法**: 对比标准 GRPO、[[PPO]]、单模块 GRPO 变体、以及不进行 prompt 优化的纯 RL 设置。借鉴了 [[BetterTogether]] 框架。
- **借鉴意义**: 直接相关。多模块 LM 程序（比如 agentic pipeline 中多次调用 LLM）的优化是实际痛点。把 GRPO 扩展到多模块 + 联合 prompt 优化，对有复杂 pipeline 的 RL 训练场景很有用。
- **锐评**: 想法不错，把 GRPO 从"一个 prompt 优化一个模型"推到"多个模块联合优化"。但实验规模偏小，只在三个数据集上做了验证，模块数量最多也就 4-5 个。实际 agentic pipeline 中模块数可以到几十个，政策梯度在多模块间的 credit assignment 问题论文基本没碰。BetterTogether 的策略直观但缺乏理论收敛性分析。👀
- **关联笔记**: [[GRPO]]（概念笔记）· [[PPO]]（概念笔记）· [[RLHF]]（概念笔记）

## 6. Invariant Learning Dynamics of Transformers in Inductive Reasoning Tasks
- **作者**: Musat, Tiberiu, Pimentel, Tiago, Zucchet, Nicolas, Hofmann, Thomas
- **机构**: ETH Zurich, Stanford University
- **链接**: [arXiv](https://arxiv.org/abs/2607.11875v1) | [PDF](https://arxiv.org/pdf/2607.11875v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 这篇做的是 Transformer 在归纳推理任务上的学习动力学理论分析。核心发现是：许多看似不同的归纳推理任务（如 induction heads 任务、重复模式识别等）共享内在的代数结构——token tuples 的上下文操纵。基于此提出 Block-List 任务类统一框架，证明 Transformer 在该类上的学习动态收敛到不变流形（invariant manifolds），揭示了 In-Context Learning (ICL) 和 In-Weights Learning (IWL) 之间的竞争关系和几何解释。还提出了 Greedy Backward Circuit Elimination 的自动电路发现方法。
- **对比方法**: 借鉴了 RASP 编程语言的形式化方法、[[LTH]]（Lottery Ticket Hypothesis）的电路分析视角。在理论框架上与 Anthropic 的 circuit 发现工作形成互补。
- **借鉴意义**: 理解 Transformer 如何做归纳推理对改进 reasoning 模型至关重要。ICL vs IWL 的竞争关系直接关系到我们在训练策略中的选择——是让模型在权重中记忆还是在上下文中推理。
- **锐评**: 好理论。从代数对称性角度理解 Transformer 推理能力的几何结构，路径很正。Block-List 任务类的统一框架让不同推理能力的比较有了共同语言。但实验只限于小型 Transformer（单层/双层 attention-only 模型），理论结果能否 scaling 到真实大小的模型是开放问题。附录里提到了 FFN 和 LayerNorm 的影响，但分析不够充分。👀

## 7. HyperSafe: Inference-Time Safety Recovery for Fine-Tuned Language Models
- **作者**: Aznaur Aliev, Carlos Hinojosa, Abdelrahman Eldesokey
- **机构**: KAUST（沙特国王科技大学）
- **链接**: [arXiv](https://arxiv.org/abs/2607.11475v1) | [PDF](https://arxiv.org/pdf/2607.11475v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: HyperSafe 解决的是微调后 LLM 安全能力退化的问题（即 safety alignment 被 fine-tuning 覆盖）。提出一个与模型无关的推理时安全恢复方法，核心是一个 activation-conditioned hypernetwork——根据输入激活动态生成安全修正信号。不需要修改原模型权重，在推理时自动检测 unsafe 输入并修正输出。在 [[LoRA]] 微调和全参数微调场景下均做了验证，在 [[AdvBench]] 和 [[BeaverTails]] 上测试。
- **对比方法**: 对比 RepNoise、SafeInstr、AEGIS、LST 等安全对齐方法。HyperSafe 的差异在于推理时恢复而非训练时对齐。
- **借鉴意义**: Fine-tuning 导致的安全退化是实际生产中的痛点。推理时安全恢复的方案比重新对齐成本低得多，直接可落地的技术。
- **锐评**: 问题选得好，方案也简洁——hypernetwork 根据激活动态生成修正比固定 safety filter 灵活。但准确率和误报率的 trade-off 没有充分讨论：误拦截合法请求的代价可能比漏拦截危险请求更隐蔽。实验只做了英文场景，多语言安全表现未知。👀

## 8. A Learning-Rate-Gated Failure of GRPO in a Small VLM Web Agent
- **作者**: Chengguang Gan 等
- **机构**: Independent Researcher, Monash University, UCAS, Shenzhen University of Advanced Technology
- **链接**: [arXiv](https://arxiv.org/abs/2607.12640v1) | [PDF](https://arxiv.org/pdf/2607.12640v1)
- **来源**: 📄 arXiv 关键词检索

![GRPO 控制实验设计](https://arxiv.org/html/2607.12640v1/x1.png)

- **核心方法**: 在严格控制的条件（固定 agent、环境、奖励、任务集）下测试 GRPO 对小模型 web agent 是否真的带来提升。结果很直接：GRPO 在大多数超参数设置下学不到东西，成功率的提升完全归因于学习率的选择——选错了 LR，GRPO 不仅不提升反而掉点。通过 interpretability 分析发现，GRPO 失败的原因是它破坏了模型已有的 SFT 知识，而 reward signal 太稀疏不足以引导模型重建行为。使用了 McNemar 检验进行统计显著性分析， [[MiniWoB]] 和 [[WebArena]] 任务集。
- **对比方法**: 对比 SFT baseline、不同学习率的 GRPO 变体、以及 GRPO + SFT 联合训练。
- **借鉴意义**: 极其相关。GRPO 是我们在 RL 训练中核心关注的算法，这篇论文直接打在了 GRPO 的痛点上——它在小模型上的有效性高度依赖超参选择，且可能摧毁已有能力。这对我们在实际训练中使用 GRPO 有重要警示意义。
- **锐评**: 这篇是今天最有价值的论文之一。负结果在某些时候比正结果更有信息量——它告诉我们 GRPO 不是万灵药，在小模型 + 稀疏奖励场景下可能适得其反。实验设计严谨（固定所有变量、统计检验、interpretability 分析），结论可信。但局限性也很明显：只试了一个 web agent 场景、一个模型大小。GRPO 在更大模型和更密集奖励下表现如何仍需验证。⚠️

## 9. EcoSpec: Cost-Aware Speculative Decoding for Mixture-of-Experts
- **作者**: Jincheng Xie, Runheng Liu, Heyan Huang, Yawen Ling, Hanbin Dai, Yu Zheng, Wen Hu
- **链接**: [arXiv](https://arxiv.org/abs/2607.12696v1) | [PDF](https://arxiv.org/pdf/2607.12696v1)
- **来源**: 📄 arXiv 关键词检索

![EcoSpec](https://arxiv.org/html/2607.12696v1/x1.png)

- **核心方法**: EcoSpec 是针对 [[MoE]] 模型设计的成本感知投机解码方案。核心观察是：MoE 模型中不同 expert 的计算成本差异很大（活跃 expert 数量 = 推理延迟的主因），现有投机解码方案忽略了这一点。EcoSpec 包括：(1) 轻量级 Expert Predictor——预测下一个 token 会激活哪些 expert；(2) Cost-Aware Draft Selection——在草稿生成时优先选择计算成本低的 expert 组合来生成 draft tokens。在 [[DeepSeek]] 等 MoE 模型上验证，相比标准投机解码提升显著。
- **对比方法**: 对比标准 [[Speculative Decoding]]、[[EAGLE-3]]、[[MTP]]（Multi-Token Prediction）。EcoSpec 的核心差异在于显式建模了 MoE 的计算成本。
- **借鉴意义**: MoE 架构已成为大规模模型的主流选择（DeepSeek、GLM-5、Mixtral 等都用了），EcoSpec 针对性的加速方案有实际部署价值。
- **锐评**: 切入点好——MoE 的投机解码确实不能简单套用 dense 模型的方案，因为 expert 路由的计算成本不可忽略。Expert Predictor 的设计轻量有效。但评估主要关注 latency，对 throughput 和 memory trade-off 的讨论不够。附录里 batch size 扩展性的分析值得看。👀
- **关联笔记**: [[Speculative Decoding]]·[[MoE]]·[[DeepSeek LLM]]

## 11. ProgramTab: Boosting Table Reasoning of LLMs via Programmatic Paradigm
- **作者**: Guo, Pei, Liu, Enjie, Tan, Yunzhi, Gao, Mochi, Zhang, Jianxin 等
- **机构**: Soochow University, Tencent
- **链接**: [arXiv](https://arxiv.org/abs/2607.11207v1) | [PDF](https://arxiv.org/pdf/2607.11207v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: ProgramTab 将表格推理分解为四个编程步骤：(1) Row Retrieval——定位相关行；(2) Column Extraction——识别相关列；(3) Data Redefinition with Code——将表格数据转化为代码友好的结构；(4) SQL Generation——生成 SQL 查询完成推理。通过程序范式替代了传统的 prompt-based reasoning，避免了 LLM 在表格理解上的格式混淆问题。在 [[WikiTQ]]、[[TabFact]]、[[HiTab]] 等标准数据集上评测。
- **对比方法**: 对比 CoT、TableCoT、ReAcTable、[[CABINET]]、OmniTab、TabSQLify、NormTab、PASTA、STAR 等表格推理方法。
- **借鉴意义**: 表格推理是教育场景（如数学解题）中的重要能力。程序化范式在需要精确计算的表格任务上可能有明显优势。
- **锐评**: 行-列-代码-SQL 的四步分解逻辑清晰，也比端到端 prompt 更可控。但多个步骤意味着错误传播——第一步 Row Retrieval 错了后续全错。同样，SQL 生成的表达能力虽强，但无法处理需要常识推理的表格问题（如"哪个年份增长最快"这种需要理解"增长"概念的查询）。在 WikiTQ 上的提升不到 2%，说服力有限。👀
- **关联笔记**: [[Llemma]]·[[MetaMath]]

## 12. Visual Access Boundaries in Vision-Language Model Reasoning
- **作者**: Osaka, Hiroto, Taniguchi, Shohei, Minegishi, Gouki, Yamashita, Kai, Suzuki, Masahiro, Matsuo, Yutaka
- **机构**: The University of Tokyo
- **链接**: [arXiv](https://arxiv.org/abs/2607.12815v1) | [PDF](https://arxiv.org/pdf/2607.12815v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 这篇研究 VLM 在推理时的"视觉注意力衰退"现象。发现 VLM 在生成过程中对图像 token 的注意力会逐渐衰减、变得嘈杂或错位。提出了 Visual Access Boundary (VAB) 来量化这个边界——即模型在推理到第几步之后开始忽略视觉信息。系统性地在 direct inference 和 [[CoT]] inference 下测量了 VAB，发现 CoT 虽然在很多任务上提升效果，但不能突破 VAB——模型的感知读出的限制成了瓶颈。在 InternVL3、[[GQA]] 等模型和数据集上验证。
- **对比方法**: 对比多种 VLM 模型（InternVL3 等）在不同推理模式下的表现。提出 DeltaCoT 等干预手段。
- **借鉴意义**: 虽然我们不做 VLM，但 VAB 现象可能在纯文本 LLM 中也有类似表现——长上下文推理时模型是否会"遗忘"开头的信息？这个分析方法可以迁移到 LLM long-context 研究。
- **锐评**: 问题定义清晰，实验设计系统化。VAB 作为一个可量化的指标很有用——让 VLM 推理中的"视觉被忽略"现象从 anecdotal 变成了可测量的。但论文主要在做诊断，缺乏有效的解决方案。感知读出限制是固有问题还是训练数据分布导致的，值得进一步研究。👀
- **关联笔记**: [[CoT]]·[[LongBench]]·[[RULER]]（长上下文评估）

## 13. Epistemic Stance Flexibility Probing: Measuring Prompt-Conditioned Register Shift in LLMs
- **作者**: Liu, Binwen, Ren, Yilin
- **机构**: Xi'an Jiaotong University
- **链接**: [arXiv](https://arxiv.org/abs/2607.12739v1) | [PDF](https://arxiv.org/pdf/2607.12739v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 提出 Epistemic Stance Flexibility Probing (ESFP) 框架，测量 LLM 根据 prompt 措辞调整认知立场（epistemic stance）的能力。用四个指标：AR（归因率，词汇层面）、PSI（措辞敏感度指数，语义层面）、SCD（立场内容密度，语用层面）、CPC（跨措辞一致性，行为层面）。测试了 [[DeepSeek]]、[[GLM-5]]、[[Claude-Sonnet]]、[[Claude-Haiku]]、[[BERT]] 等模型。
- **对比方法**: 借鉴了 MAS（Metacognition Assessment Scale）临床心理学的元认知评估方法。
- **借鉴意义**: LLM 的"立场灵活性"直接关系到模型在教育和辅导场景中的表现——是否能根据学生的理解水平调整解释方式。
- **锐评**: 跨学科视角有意思——把临床心理学的元认知评估方法引入 LLM 评估。四个指标从词到语用的多层设计比较全面。但样本量偏小（只测了少数模型和少数 phrasing 变体），且 framing 本身是否真的能反映元认知能力还是只是 prompt sensitivity 的代理指标，存疑。⚠️

## 14. Elenchos: Evaluating Abductive Reasoning with LLMs
- **作者**: Julius Steiglechner, Lucas Mahler, Gabriele Lohmann
- **机构**: Max Planck Institute for Biological Cybernetics
- **链接**: [arXiv](https://arxiv.org/abs/2607.12733v1) | [PDF](https://arxiv.org/pdf/2607.12733v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: Elenchos 是一个评估 LLM 溯因推理能力的框架。核心设计是"结构反转"：把形式系统本身当作诊断对象，通过构造突变 (mutation) 来生成不同难度的推理问题。采用 Difficulty Ladder 设计（逐步增加复杂度），以及 Wide-Panel Benchmark（多模型多难度比较）。涵盖类标签映射、简单反射 agent、非退化过滤等变异分类。
- **对比方法**: 对比标准静态推理 benchmark（MMLU 等）的评估方式。Elenchos 的差异在于通过突变生成可控难度的推理链。
- **借鉴意义**: 推理评测是核心关注方向。Elenchos 的"难度梯子"设计比传统的静态 benchmark 更能精细地区分模型的推理能力边界。
- **锐评**: 框架设计有巧思——突变生成问题比手动构造更系统化也更可扩展。但论文目前的实验设置偏理论化（class label mapping 等抽象任务），与现实推理场景的差距较大。LambdaPy 的语法设计也限制了评测范围。作为评测框架值得关注，但实际评估结论还需要在更现实的任务上验证。👀

## 18. The Paternalistic Filter: Epistemic Injustice in LLM-Mediated History Education
- **作者**: Alexis Popovici, Andrei Ionașcu, Adrian Marius Dumitran
- **链接**: [arXiv](https://arxiv.org/abs/2607.11292v1) | [PDF](https://arxiv.org/pdf/2607.11292v1)
- **来源**: 📄 arXiv 关键词检索

- **核心方法**: 研究 LLM（如 ChatGPT）在为罗马尼亚少数族裔学生提供历史教育时的"父爱式过滤"现象。通过 persona-based 实验设计，发现 LLM 对不同身份背景的学生给出不同的回答——对边缘化群体的回答更简略、更规避敏感话题。测量指标包括 Differential Refusal Rate（差异化拒绝率）、Type-Token Ratio（词汇多样性）、Justification Scores（理由充分性得分）。
- **对比方法**: 采用了心理学和社会学的研究方法（persona-based 实验、内容分析）。
- **借鉴意义**: 教育领域密切相关。LLM 在教育场景中可能存在系统性偏见，对少数群体学生的公平性有潜在影响。这对我们做教育相关的 LLM 应用有警示意义。
- **锐评**: 挖了一个很重要的坑——LLM 在教育中不仅是"回答得对不对"的问题，还有"对谁说了什么"的公平性问题。persona-based 实验设计合理。但样本量小（只针对罗马尼亚语境），论文的方法论部分略显薄弱。作为 raise awareness 的工作有意义，但从解决方案角度看提得不多。⚠️
